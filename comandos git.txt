git init (Criar uma pagina git)

git status(Verificar oque há dentro da pagina)

git add .(adicionar todos elementos dentro da pagina) 

git commit -m ""(Vesionar o codigo e colocar mensagem) 

git push add origin (Colocar o repositorio do github) 

git push(Subir o projeto para o repositorio do github) 

git reflog (Ver historico e id do projeto)

git reset --hard + o id do projeto (Voltar as versões anteriores)

git branch + o nome da branch (Criar clones para testar o codigo)

git checkout + nome da branch (Mudar o projeto principal)

git pull (Trazer todas as atualização do servidor) // primeiro usar esse comando antes do git merge

git merge + nome da branch (Juntar código do projeto secundario com o primario)




1-Git pull na branch principal;
2-Gerar uma nova branch a partir da branch principal;
3-Trabalhar e adicionar novas funcionalidade na nova branch que criou;
4-Finalizar o trabalho na branch secundaria;
5-Git checkout na branch principal;
6-git pull;
7-git merge;
8-git push na branch principal;
